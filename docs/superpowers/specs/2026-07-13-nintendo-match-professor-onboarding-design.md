# 브레인워크 — 닌텐도급 교수 개입·온보딩·납득성·손맛 맞춤 설계

- 날짜: 2026-07-13
- 상태: 설계 승인 완료 (구현 대기)
- 레퍼런스: **닌텐도 DS 「매일매일 두뇌 트레이닝」(2005, 카와시마) 클래식 구조**
- 교수 페르소나: 카와시마 자리에 **말티즈 박사**(기존 mascot.png 재사용). 기본 이름 **"월크 박사(Dr. Walk)"** — spec 리뷰에서 변경 가능
- 용도: **내부용**(상업 배포·IP 심의 대상 아님). 단 "재미로 보는 뇌 나이" 의료 면책은 유지

## 목표

닌텐도 원조 수준으로 **① 교수가 처음부터 끝까지 개입**하고, **② 온보딩을 교수가 손잡고 협조**하며, **③ 뇌 나이 결과가 누가 봐도 납득**되고, **④ 미니게임 손맛이 원조급**이 되도록 프로그램 구조를 맞춘다.

## 정직한 베이스라인 (이미 있는 것 — 재빌드 금지)

구현을 시작하기 전에, 아래는 **이미 존재**하므로 새로 만들지 않는다:

- **데일리 허브**: 도장 달력, 오늘의 훈련, 뇌 나이 체크(1일1회), 점수 그래프 — 원조와 근접 (`renderStats`, `renderHome`)
- **고정 체크 배터리**: 5과제 시간제 셔플 (`CHECK_POOL`, `startCheck` app.js:924)
- **뇌 나이 산식**: 속도나이(과제별 RT 앵커 `BEST_RT`)+오답가산+EMA 블렌딩 (`finishCheck` app.js:824-856)
- **결과 투명화**: 평균 반응·정답률·과제별 내역 노출 (finishCheck 이후)
- **미니게임 손맛 레이어 대부분**: `FX.flash(ok)`가 **모든 정답/오답이 지나는 공통 길목**(juice.js:74)이며 이미 닌텐도식 ⭕/✕ 판정 오버레이(`FX.judge`), 탭 위치 스파크(`FX.spark`), 콤보 배지·피버 오라·콤보버스트, 진동(navigator.vibrate), 효과음(SND.good/bad 피치 상승)을 포함
- **교수 얼굴 포지션**: mascot base/happy/sad 표정 스왑 (`setCoachFace` app.js:632, 주석 "원작 교수 얼굴 포지션")
- **코치 대사 풀**: `COACH`(record/high/mid/low) + `coachSay` (app.js:620-630)

→ **순 신규 작업은 ①②③⑤ 콘텐츠와 훅, ⑥은 신설이 아닌 튜닝·일관성 점검이다.**

## 6축 상세 설계

### ① 말티즈 교수 인트로 + 온보딩 협조 (신규)

**현재**: first-run = (선택)이름 입력 → 코치마크 투어가 **훈련부터** 안내.
**변경(닌텐도식)**: first-run 순서를 **교수 인트로 → 첫 뇌 나이 측정 → 결과 → 허브 안내**로 재배치.

- 온보딩 인트로 화면(기존 화면 스위처/인트로 구조 재사용 우선, 부족하면 신규 `screen-onboard` 추가):
  - 월크 박사 등장 대사(원조 톤): "안녕하세요, 월크 박사입니다. 두뇌는 쓸수록 젊어지죠. 먼저 당신의 뇌 나이부터 재볼까요?"
  - 단일 CTA: **"뇌 나이 재기"** → `startCheck` 직행
  - 이름은 여기서 자연스럽게 요청(선택). 측정 결과에 이름 남김
- first-run 판별: 기존 그랜드파더링 조건 재사용(`history().length || ageChecks().length` 없으면 신규). 기존 사용자는 인트로 스킵
- **투어 재배치**: 기존 `TOUR` 5스텝(app.js:281-287)을 **측정 완료 후** 시작하도록 순서 변경. 1스텝 문구를 "이제 매일 훈련해 젊게 유지해요 — 오늘의 훈련부터!"로. 즉 측정이 온보딩의 첫 걸음, 훈련이 두 번째
- 교수가 **투어 내내 개입**: 각 코치마크 말풍선 앞에 교수 페르소나 프레이밍(말티 박사 아이콘+교수체)

### ② 첫 부팅 = 뇌 나이 측정 직행 (기존 재사용)

- ①의 인트로 CTA가 곧바로 `startCheck` 실행. 신규 게임 로직 없음
- 측정 브리핑(#intro-desc app.js:511)에 교수 목소리 얹기(③과 통합)

### ③ 교수 상시 개입 — 닌텐도 수준 (신규 대사 풀 + 훅 확장)

교수가 세션 전 구간에 등장. 기존 `coachSay`/`#coach-bubble`/`setCoachFace` 위에 **장면별 대사 세트**를 신설:

- **홈 인사**: 시간대별(아침/오후/저녁) + 연속출석 반응("3일째 산책 중이시네요!"). `renderHome`에 훅
- **체크 시작 브리핑**: "긴장 마세요. 빠르고 정확하게, 그거면 됩니다." (#intro-desc 대체/보강)
- **측정 중**: 과제 전환 시 짧은 응원(선택, 리듬 안 깨는 선에서)
- **공개 의식**: 기존 "두구두구두구…"(app.js:862) 유지 + 공개 직후 **과제별 촌평**(⑤와 통합)
- **훈련 후**: 기존 coachSay 유지하되 교수체로 톤 통일
- 페르소나 규약: 말투는 정중+따뜻한 교수체, 강아지 유머는 소량 유지(친근감). 표정은 setCoachFace 재사용
- 대사는 데이터 상수(`PROF` 객체)로 분리해 유지보수 용이

### ④ 데일리 허브 (정렬만, 재빌드 X)

- 이미 원조 근접. 변경 최소:
  - 홈에서 **뇌 나이 체크**를 시그니처 액션으로 시각적 강조(원조는 매일 측정이 중심)
  - 교수 인사 배치(③)
- 마일/해금 경제는 손대지 않음(이미 도장 기반)

### ⑤ 뇌 나이 납득성 — 누가 봐도 수긍 (신규 내러티브)

거짓 정밀도가 아니라 **투명성+교수 내러티브**로 납득을 만든다. "재미로 보는" 면책은 유지.

- **교수 진단 내러티브(핵심)**: 과제별 강약을 말로 풀어 개인화된 근거 제시.
  - 예: "계산은 30대처럼 빨랐지만, 기억 과제에서 시간이 걸렸어요. 그래서 종합 XX세로 봅니다."
  - 구현: `finishCheck`의 `metas`(과제별 ratio·err 이미 계산됨 app.js:828-836)에서 **가장 빠른/느린 과제, 최고/최저 정답 과제**를 뽑아 문장 생성
- **"왜 이 나이인가" 분해 카드**: 속도 기여분 vs 오답 기여분을 시각적으로(막대/비율). speedAge와 errAdd가 이미 분리돼 있음(app.js:838-839) → 그대로 노출
- **또래 프레이밍**: "당신 또래 평균 대비 ±N세" 식 그럴듯한 노멀 곡선(실측 아님, 표기 명확히). 과장 없이 상대 위치만
- **가드**: 여전히 20~85세 범위·EMA 유지. 의료 판정·치매 위험 문구 절대 금지(기존 법적 가드레일)

### ⑥ 미니게임 손맛 — 튜닝·일관성 점검 (신설 아님)

`FX.flash` 공통 레이어가 이미 강력하므로 **감사+튜닝**만:

- **전 게임 일관성 감사**: 모든 미니게임이 정답/오답 판정을 `FX.flash(ok)`로 통과시키는지 점검(누락 게임은 연결). 누락되면 그 게임만 ⭕✕·스파크·진동·콤보가 빠져 손맛 불균일
- **튜닝**: judge 오버레이 지속/투명도, 스파크 수, 진동 패턴이 원조처럼 "짧고 경쾌"한지 실기기 감으로 조정
- **정답 도장(⭕) 강화**: 원조의 빨간 동그라미 손맛에 맞춰 judge-o 연출 소폭 강화(옵션)
- **신규 코드 최소** — 대부분 기존 상수·CSS 조정

## 페르소나 스펙 (월크 박사)

- 이름: 월크 박사(Dr. Walk) — 변경 가능
- 종: 말티즈(기존 에셋)
- 말투: 정중·따뜻한 교수체 + 강아지 유머 소량. 예: "~볼까요?", "제가 재드리죠", "킁킁, 실력 냄새가"
- 표정: base/happy/sad (setCoachFace 재사용)
- 새 일러스트(안경·가운)는 **후순위** — 우선 mascot.png + 텍스트 프레이밍(🎓)

## 범위 제외 (YAGNI)

- **음독(소리내어 읽기)** 게임 — 마이크·음성인식 필요, 제외
- 마일/해금 경제 재설계 — 손대지 않음
- 새 교수 일러스트/신규 미니게임 — 후순위 후보
- 실측 또래 노멀 데이터 — 그럴듯한 곡선으로 대체(표기 명확)

## 구현 순서 (한 번에 전부, 단 안전 순서)

1. `PROF` 대사 상수 + 교수 페르소나 헬퍼(③) — 이후 축들이 이걸 씀
2. 온보딩 재배치(①②): 인트로 화면 → startCheck 직행 → 투어 측정 후로
3. 납득성 내러티브·분해 카드(⑤): finishCheck 확장
4. 홈 교수 인사·체크 강조(④)
5. 손맛 감사·튜닝(⑥): FX.flash 커버리지 점검 + 상수 조정
6. 캐시버스트 `?v=59→60` (모든 편집 끝난 커밋 직전 마지막)

## 검증

- 로컬 서버(launch.json "brain-walk")로 브라우저 E2E:
  - 신규 유저 시뮬(localStorage 비우고): 인트로→측정→결과 내러티브→허브→투어 순서 확인
  - 기존 유저: 인트로 스킵 확인(그랜드파더링)
  - 측정 결과: 방치=고령/빠름+무오답=젊음 극단값 여전히 정상(회귀)
  - 손맛: 각 미니게임 1판씩 ⭕✕·스파크·진동 나오는지(FX.flash 커버리지)
- `sim/autoplay.js` 하네스 재실행: 25종+데일리+체크 E2E 무회귀, 콘솔 에러 0
- 콘솔 에러 0 확인 후 커밋. push=자동배포이므로 사용자 승인 후

## 개발 함정 (기존 노트 준수)

- 캐시버스트는 **마지막**에(범프 후 파일 수정 금지)
- prompt/confirm/alert 목킹 후 클린 리로드
- 다크모드: 배경색 고정 금지, 테마 변수만
- 데일리 라인업 랜덤 유지(날짜시드 복원 금지)
- 게임 추가 시 sim 솔버도 추가(이번엔 신규 게임 없음)
